# 새 PC 셋업 — 2줄로 끝

CEO 님이 새 컴퓨터를 잡으셨거나, 기존 PC 에 클로드 코드를 처음 까셨을 때
**평생 딱 한 번** 하시면 되는 셋업입니다.

---

## ⚡ 잠깐 — 새 *프로젝트* 시작하는 거면 더 빠른 길이 있습니다

**기존 PC + 새 프로젝트** 라면 이 문서 *건너뛰셔도* 됩니다:

1. https://github.com/1215kkm/Ai_Team → **"Use this template"** → 새 레포 이름 → Create
2. 그 새 레포에서 클로드 코드 열고 → `/회의시작`

→ `curl` 칠 일 0번. 강팀 풀세트가 처음 커밋부터 들어있음.

> 이 문서(아래 내용)는 *PC 자체에 강팀을 영구 설치*하는 경우용입니다.
> 기존 레포에 "강팀 불러와" 하기 위해 PC 글로벌 셋업이 필요할 때.

---

## 사전 준비

- 클로드 코드 데스크탑 앱 또는 CLI 설치 — https://claude.com/code
- Windows 면 추가로 **Git for Windows** 설치 (Git Bash 가 같이 깔립니다) — https://git-scm.com/download/win
- 텔레그램 봇 토큰·chat_id (이미 다른 PC 에서 만드셨으면 그대로 재사용 — 새로 만드실 필요 없음)

---

## 1단계 — 터미널 열기

| OS | 어디서 |
|---|---|
| **Windows** | 시작 메뉴 → "**Git Bash**" 검색 → 실행 |
| **Mac** | `Cmd + Space` → "터미널" → 실행 |
| **Linux** | 평소 쓰시는 터미널 |

> 폴더는 어디든 상관없습니다. 홈 디렉토리(`~`) 그대로 두셔도 됨.

---

## 2단계 — 두 줄 붙여넣기

터미널에 이 두 줄을 *순서대로* 붙여넣고 엔터:

```bash
curl -fsSL https://raw.githubusercontent.com/1215kkm/Ai_Team/main/scripts/pull-team.sh | bash -s -- --no-local
bash ~/.claude/bin/setup-telegram.sh
```

### 첫 줄 — 강팀 부트스트랩
- GitHub 의 강팀 본체에서 5명 에이전트·슬래시 커맨드·디자인 시스템·스크립트를 받아 `~/.claude/` 에 박습니다.
- `--no-local` 플래그 덕에 *현재 폴더는 안 건드림* — 새 PC 첫 셋업용.
- 끝에 "✅ 강팀 도착" 뜨면 성공.

### 두 번째 줄 — 텔레그램 연결
물어보는 두 가지에 답하시면 됩니다:
1. **봇 토큰** — BotFather 에서 받은 문자열 (예: `123456:ABC-DEF...`)
2. **chat_id** — 봇과 한 번이라도 텔레그램에서 대화했으면 *자동 감지*. 안 잡히면 직접 입력.

> 다른 PC 에서 이미 셋업하신 적 있으면 *같은 토큰·같은 chat_id* 그대로 입력하시면 됩니다. 그러면 모든 PC 의 회의록이 *같은 텔레그램 채팅* 으로 들어옵니다.

마지막에 "✅ 강팀 → 텔레그램 연결 성공" 메시지가 텔레그램에 오면 끝.

## 3단계 — 사용 (PC당 평생 0번 더, 작업할 때마다 1번)

이제 어떤 레포에서 클로드 코드 켜시든 채팅창에:

```
강팀 불러와 회의 시작해
```

→ 자동으로:
1. 그 레포에 강팀 박음 (10초)
2. 강팀장이 안건 자동 생성 → 5명 회의
3. 결정·미결정·다음 액션 정리
4. **`📦 그_레포_이름 · 🎯 회의_주제`** 헤더 박힌 보고서를 텔레그램으로 발송

## 자주 묻는 것

### Q. PC 두 대 쓰면 봇 새로 만들어야 하나?
A. **아니요.** BotFather 토큰·chat_id 둘 다 그대로 재사용. 모든 PC 회의록이 한 채팅으로 모입니다.

### Q. 강팀 본체가 업데이트되면 자동으로 따라오나?
A. 새 레포에 `강팀 불러와` 할 때마다 *항상 main 최신*을 받습니다. 이미 강팀이 박힌 레포에 최신 적용하려면 그 레포에서 다시 `강팀 불러와`.

### Q. 데스크탑 앱에서 했는데도 *클라우드 컨테이너* 라고 하면 어떻게?
A. 클로드 코드의 *세션 모드*에 따라 다릅니다. 진짜 PC 로컬 셸을 쓰는 *Claude Code CLI* (`claude` 명령어) 모드면 영구. 클라우드 컨테이너 모드면 컨테이너 종료 시 사라집니다. 자세한 건 [환경 확인하기](./환경-확인.md) (추가 예정).

### Q. 명령어 어렵다 — 그냥 클로드한테 시키면?
A. 됩니다. 클로드 코드 채팅창에 그대로:
```
이거 한 번 돌려줘:
curl -fsSL https://raw.githubusercontent.com/1215kkm/Ai_Team/main/scripts/pull-team.sh | bash -s -- --no-local
이거도 돌려줘:
bash ~/.claude/bin/setup-telegram.sh
```
→ 권한 승인 한 번 누르시면 끝.

> 단, *이 세션이 클라우드 컨테이너인지 진짜 로컬인지* 에 따라 영구냐 일회용이냐가 갈립니다. 클로드한테 `pwd` 와 `hostname` 한번 쳐달라고 하면 확인 가능.

## 실패하면

| 증상 | 해결 |
|---|---|
| `curl: command not found` (Windows PowerShell) | Git Bash 로 옮겨서 다시 실행 |
| `404 Not Found` | 강팀 본체 PR 이 main 머지 안 됐을 가능성. https://github.com/1215kkm/Ai_Team/tree/main/scripts 에 `pull-team.sh` 있는지 확인 |
| 텔레그램 chat_id 자동 감지 실패 | 봇과 텔레그램에서 *아무 메시지나 한 번* 보낸 뒤 다시 실행 |
| 권한 거부 (`Permission denied`) | `chmod +x ~/.claude/bin/*.sh` 한 번 돌리고 재시도 |
| 회의는 되는데 텔레그램 안 옴 | `bash ~/.claude/bin/setup-telegram.sh` 다시 돌려서 토큰 재입력 |

## 빈 GitHub 레포에서 "강팀 불러와" 가 안 먹힐 때

증상: 새로 만든 빈 레포를 *클라우드 컨테이너* 클로드 코드 세션으로 열고 "강팀 불러와" 라고 쳤더니 클로드가 *"강팀이 뭔지 모르겠다"* 라고 되묻는다.

원인: 그 컨테이너의 글로벌 `~/.claude/CLAUDE.md` 가 비어 있어 트리거 룰이 *어디에도 없음*. 자연어로는 인식 불가.

해결 — **레포 README 첫 줄에 부트스트랩 명령을 박는다** (클로드가 새 세션에서 가장 먼저 읽는 파일):

```markdown
> 이 레포는 강팀(Ai_Team) 5명짜리 AI 팀이 운영합니다.
> 클로드 코드 세션 첫 메시지로 채팅창에 붙여넣으세요:
>
>     이거 한 줄 돌려줘: curl -fsSL https://raw.githubusercontent.com/1215kkm/Ai_Team/main/scripts/pull-team.sh | bash
```

이러면 빈 레포에 들어간 클로드도 *README 만 보고* 부트스트랩을 알아서 돌립니다. `scripts/new-project.sh` 로 만든 레포는 이 줄이 *이미* 자동으로 박혀 있음.

> 다른 사람한테 강팀 박힌 레포를 공유할 때도 README 에 이 한 줄이 있으면 그쪽 PC 의 클로드가 *세팅 없이* 바로 강팀을 박을 수 있습니다.
