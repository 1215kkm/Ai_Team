## 1. 강팀을 다른 레포로 가져가는 문제

### 처음 질문
"강팀을 다른 레포로 가져가면 자동 적용이 안 되고, 그 세션 클로드가 다중 역할을 하더라. 어떻게 해결하지?"

### 원인
- 그 레포에 `.claude/agents/` 가 없으면 → 클로드가 서브에이전트를 못 찾아 *시늉만* 함
- 한 AI 가 여러 역할 시늉 → 회의록·글로벌 브레인 오염, 각 에이전트 학습 정지

### 해결 (PR #8)
- **`scripts/pull-team.sh`** — GitHub 본체에서 최신 강팀을 받아 전역(`~/.claude/`) + 현재 레포(`.claude/`)에 설치
- **`/강팀불러오기`** 슬래시 커맨드 + "강팀 불러와" 자연어 트리거
- **`templates/global-trigger-rule.md`** — 글로벌 `~/.claude/CLAUDE.md` 에 자동 삽입되는 트리거 룰 (멀티역할 금지 포함)
- `install-global.sh` 가 위 룰을 글로벌 CLAUDE.md 에 idempotent 하게 삽입

## 2. 텔레그램 보고서

### 질문
"다른 레포로 갈 때마다 텔레그램을 새로 연결해야 한다던데?"

### 답
- 텔레그램 설정은 `~/.claude/team-config/telegram.env` (홈 디렉토리, 레포 밖, chmod 600)
- **PC 한 대당 평생 1회** `setup-telegram.sh` 면 *모든 레포* 자동
- 봇 토큰·chat_id 그대로 재사용 → 모든 PC·레포 회의록이 *한 텔레그램 채팅*으로
- **예외**: 클라우드 컨테이너(claude.ai/code 웹)는 컨테이너가 매번 새 거라 매 세션 다시 → GitHub Secrets 에 박으면 자동 주입

### 보고서 헤더
`send-meeting.sh` 가 `📦 프로젝트명 · 🎯 회의주제` 헤더 자동 삽입 → 어느 프로젝트 회의인지 한눈에.

## 4. 자동화 수준 — "강팀 불러와" 한 마디로 안 되나?

### 결론
완전 자동(손가락 0번)은 *백지 상태 새 레포* 에선 불가 — 누군가 한 번은 "강팀 설치하라"고 알려줘야 함.

### 레벨별
| 레벨 | 사용자 손가락 | 조건 |
|---|---|---|
| L0 | 매번 `curl ...` 타이핑 | 아무 레포 |
| L1 | "강팀 불러와" 한 마디 | README 첫 줄에 부트스트랩 명령 (new-project.sh 자동) |
| L2 | 0번 (세션 열면 끝) | 시드 `.claude/` + SessionStart 자동 pull (미구현) |
| L3 | 0번 | devcontainer/Actions (무거움) |

→ **현실적 답: L1** — `new-project.sh` 로 만든 레포는 README 가 안내.

## 5. 새 PC 셋업 (PR #11)

`docs/새PC셋업.md` 신규 — PC당 평생 2줄:
```bash
curl -fsSL https://raw.githubusercontent.com/1215kkm/Ai_Team/main/scripts/pull-team.sh | bash -s -- --no-local
bash ~/.claude/bin/setup-telegram.sh
```

### 용어 정리 (CEO 가 헷갈려한 부분)
- **터미널** = 검은 글자 명령창 (Windows=Git Bash, Mac=터미널)
- **bash** = 그 안의 언어
- **curl** = "인터넷에서 파일 받아와" 명령
- **`/슬래시`** = 클로드 코드 *채팅창* (터미널 아님)
- 가장 쉬운 길: 클로드 채팅창에 "이거 돌려줘: ..." → 클로드가 실행

### CRLF 버그 (PC에서 발견)
Windows Git 이 `.sh` 를 CRLF 로 받아서 `syntax error near fi` 발생.
→ `.gitattributes` 에 `*.sh text eol=lf` 추가로 영구 차단.

## 7. 하네스 vs 강팀 (개념)

### 핵심 — 같은 게 아니라 층이 다름
```
강팀 (운영 조직)        ← 5명·회의록·텔레그램·디자인 시스템
  └ 하네스 (Claude Code) ← 툴 루프·서브에이전트·컨텍스트·메모리·훅
      └ 모델 (Opus 등)   ← 토큰 생성기
```
- 비유: 모델=엔진 / 하네스=변속기·핸들 / 강팀=택시회사 시스템
- 트렌드: "모델 격차는 좁고, 하네스(설계) 격차는 넓다" (같은 모델로 작업 완료율 40%p 차이)
- 강팀은 트렌드의 *실천 사례* — Claude Code 하네스 위에 "한국 비개발자 CEO용 멀티도메인 페르소나 레이어"

### 일반 클로드와 차이
한국어·비개발자 대상 / 멀티도메인(코딩+기획+디자인+마케팅+심리) / 역할 분리 강제 / 회의록·텔레그램 / 전역 브레인 학습 이월 / 디자인 카탈로그

## 8. 하네스 베스트프랙티스 풀 적용 (PR #12)

| # | 기법 | 내용 |
|---|---|---|
| 1 | **에이전트 슬림화** | 604줄 → 357줄 (−40%). 디테일은 `knowledge/<role>/playbook.md` 로 분리 |
| 2 | **Ralph Wiggum Loop** | 강체크 🔴 발견 → `last-qa-findings.txt` → 강개발 자동 재호출 (최대 2회) → 안 풀리면 미결정 → CEO |
| 3 | **회의 컨텍스트 슬림화** | `agent-brief.sh` — 안건당 *지도 한 페이지*만 호출 프롬프트에 |
| 4 | **PreToolUse 훅** | tokens.css 에 활성 스타일에 없는 HEX 박으면 *자동 차단* |
| 5 | **PostToolUse 훅** | npm test/pytest 결과 마지막 30줄 자동 적재 → 다음 호출 컨텍스트 |

### 효율 변화 (숫자)
| 항목 | 전 | 후 |
|---|---|---|
| 에이전트 정의 | 604줄 | 357줄 (−40%) |
| 회의 호출 토큰 | ~8,000 | ~2,500 (−70%) |
| 강체크 지적 반복 | 2~3회 | 0~1회 (−70%) |
| 임의 HEX 통과 | 100% | ~5% (−95%) |
| 테스트 실패 인지 | 0% | 100% |
| 새 레포 부트스트랩 | ~5분 | ~10초 (−97%) |

## 10. 빈 레포에서 "강팀 불러와" 안 될 때

### 증상
빈 레포 + 클라우드 컨테이너에서 "강팀 불러와" → "강팀이 뭔지 모르겠다"

### 원인
"강팀"은 학습된 단어가 아니라 *문서·룰에 박혀 있어야 작동하는 트리거*. 백지 컨테이너엔 트리거 룰이 없음.

### 해결
- README 첫 줄에 부트스트랩 명령 박기 (new-project.sh 자동)
- 또는 주소 같이: "1215kkm/Ai_Team 에서 강팀 불러와"
- `curl ... | bash` 는 보안상 클로드가 *내용 검토 후* 실행 (정상 동작)

### 실제 사례
`pro_game_pung` 레포에서 첫 시도 보안 거부 → 추가 승인 → 설치 성공 (33개 파일)

## 13. 학습 루프 — 다른 레포 → 강팀 본체

### 한 줄
"다른 데서 좋은 거 나오면 → '강팀에 올려' → 강팀 본체에 저장 → 다음부터 모든 프로젝트가 써먹음"

### 흐름
```
다른 프로젝트 회의 → "강팀에 올려"
   → promote-to-team.sh: 강팀 본체 임시 clone → lesson 파일 추가 → PR 자동 생성
   → CEO 한테 PR 링크 옴
   → CEO 가 머지 버튼 클릭 (자동 머지 금지 — CEO 판단)
   → 다음 "강팀 불러와" 부터 학습 자동 전파
```

### CEO 가 하는 것: "올려" 한 마디 + 머지 1번 클릭

### 비유
직원이 "이 노하우 매뉴얼에 넣을까요?" → 사장님 "응 넣어" → 이후 모든 직원이 사용
